WiCyS / SANS Beginner-Level CTF

This event marked the beginning of my structured CTF documentation framework. Rather than focusing solely on flag submission, each challenge is treated as a mini investigation emphasizing repeatable workflows, disciplined pivoting, and evidence-to-conclusion justification.

Investigation approach

  • Observation — Identify anomalies in the provided artifacts
  • Hypothesis — Form theories about what happened and why
  • Testing — Validate ideas using tools and targeted checks
  • Evidence — Capture outputs, key artifacts, and supporting notes
  • Conclusion — Make defensible claims tied directly to evidence

Tools used

  • Linux command line
  • Network analysis tooling (as applicable)
  • Log analysis techniques
  • Basic forensic investigation workflows

Example investigation: Suspicious authentication activity

During one challenge, the objective was to determine the source of unauthorized authentication attempts observed in system logs. The workflow focused on pattern recognition, correlation across related events, and narrowing likely sources.

Process

  • Review authentication logs for unusual patterns and outliers
  • Correlate timestamps between related events to build a timeline
  • Identify abnormal login sources and repeated access attempts
  • Validate findings by comparing expected vs. observed behavior

Outcome

The investigation surfaced repeated authentication attempts originating from an external source, consistent with a brute-force access pattern.
